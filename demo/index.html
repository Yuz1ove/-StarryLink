持續追蹤

多路徑通訊決策矩陣 channelScore 
channelScore =
packetSuccessRate * 0.35 +
latencyScore * 0.20 +
signalStrength * 0.15 +
gpsAvailability * 0.15 +
channelCost * 0.10 +
batteryImpact * 0.05

channelCost 分數越高代表越適合使用，不是越昂貴。一般狀態 Wi-Fi / 5G / SMS 優先；Red 高風險且地面網路失敗時，Satellite Backup 才提高優先級。

依封包成功率、延遲、訊號、GPS、成本與電量影響加權選路。

通訊路徑 總分 為什麼 packetSuccessRate latencyScore signalStrength gpsAvailability channelCost batteryImpact score breakdown
風險分數不只決定救援優先順序，也會影響通訊路徑選擇。 Green / Yellow：使用 Wi-Fi、5G、SMS。 Orange：啟用低資料量封包，增加重送頻率。 Red：優先傳送 GPS、求救狀態、生命狀態，必要時升級衛星備援。

function calculateRiskScore(userState) {
  rawRiskScore = selectedButtons + reply + gps + retry + battery + medical + lastSync
  displayRiskScore = clamp(rawRiskScore, 0, 100)
  return { rawRiskScore, displayRiskScore }
}

function getRiskLevel(score) {
  if (score >= 80) return "RED"
  if (score >= 55) return "ORANGE"
  if (score >= 25) return "YELLOW"
  return "GREEN"
}

function calculateChannelScore(channelState, userRisk) {
  return weightedScore(channelState) + riskAdjustment(userRisk)
}
