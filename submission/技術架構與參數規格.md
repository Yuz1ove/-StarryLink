# 星夜守望者 技術架構與參數規格

## 可展示 MVP

展示流程：建立事件 -> 取得網路狀態 -> 計算通道分數 -> 產生低頻寬訊息 -> 送出主通道與備援通道 -> 追蹤回覆率。

## 系統模組

1. 使用者端
   - 手機 App / Web 入口 / 簡訊互動 / 語音互動
   - 功能：一鍵報平安、緊急求助、家庭或團隊狀態面板

2. 事件 API
   - `POST /events`
   - `POST /route/plan`
   - `POST /notify/send`
   - `POST /ack`
   - `GET /events/{id}/dashboard`

3. AI 編排層
   - 事件分級
   - 訊息摘要
   - 低頻寬 payload 改寫
   - 通道選擇
   - RAG SOP 查詢

4. 通訊服務層
   - Push
   - SMS
   - Voice IVR
   - Satellite Brief
   - Fixed Broadband
   - Microwave / Enterprise Backup

5. 營運管理台
   - 通道分數
   - 送達率
   - 回覆率
   - 未確認名單
   - SLA 倒數
   - 備援啟用原因

## 事件資料模型

```json
{
  "event_type": "typhoon_family_check",
  "severity": 4,
  "location_risk": 78,
  "recipient_count": 12,
  "ack_target": 0.8,
  "privacy_level": "emergency_minimum",
  "battery_level": 42,
  "network_snapshot": {
    "bandwidth_kbps": 48,
    "latency_ms": 950,
    "packet_loss": 18,
    "channel_availability": {
      "cellular": true,
      "fixed": false,
      "satellite": true,
      "enterpriseBackup": false
    }
  }
}
```

## 通道評分公式

```text
score = 0.30 * availability
      + 0.20 * signal
      + 0.15 * latency_score
      + 0.15 * loss_score
      + 0.10 * cost_score
      + 0.10 * policy_match
```

判斷：

- `score >= 75`：主通道候選
- `60 <= score < 75`：可作主通道，但必須啟用備援
- `45 <= score < 60`：只作備援
- `score < 45`：不建議使用

## 低頻寬 payload 規則

- `bandwidth_kbps >= 256`：完整訊息，可含地圖連結與說明
- `64 <= bandwidth_kbps < 256`：精簡訊息，保留事件、地點、回覆選項
- `16 <= bandwidth_kbps < 64`：160 字以內，適合 SMS
- `bandwidth_kbps < 16`：狀態碼 + 座標 + 時間 + 回覆碼

## SLA 參數

- 家庭平安回報：5 分鐘內取得 80% 回覆
- 企業據點：10 分鐘內取得 90% 值班確認
- 社區公告：30 分鐘內產出未確認名單
- 戶外巡檢：3 分鐘內完成求助送達或啟動衛星備援

## Demo 情境

1. 颱風夜家庭平安回報
   - 弱網、行動網路可用、固網不可用
   - 預期：SMS 主通道，Push 與 Voice 備援

2. 企業據點斷網
   - 固網中斷、企業備援可用、群組人數較多
   - 預期：Microwave / Enterprise Backup 主通道，SMS 備援

3. 山區巡檢弱訊號
   - 頻寬極低、延遲高、衛星可用
   - 預期：Satellite Brief 主通道，SMS 備援
